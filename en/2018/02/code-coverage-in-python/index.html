Code Coverage in Python

Lucas Xu / 2018-02-23

What is Code Coverage?

In short, Code Coverage testing is determining how much code is tested.

Test coverage is a measured used to describe the degree to which source code of a program is executed when a particular test suits runs. — Wikipedia Code coverage - Wikipedia

Why Test Code Coverage?

Code coverage testing helps you to figure out how well your tests actually test your code. Especially when the code base is large, it lets you have a clear understanding of how well your tests have been written. Furthermore, it helps you maintain the test quality over the lifecycle of your project.

Code Coverage in Python

A popular tool for testing the code coverage in Python is Coverage.py. It works for both Python2 and Python3. More details: Coverage.py — Coverage.py 4.5.1 documentation

Moreover, some test runners provide coverage integration to make it easy to use coverage.py while running tests. A good example is pytest-cov. More details: Welcome to pytest-cov’s documentation! — pytest-cov 2.5.1 documentation

Based on the pytest-cov official documentation, the benefits of using the plugin rather than just coverage run via using Coverage.py directly, is first, it supports subprocess well, second, it supports Xdist. Most importantly, if you are using pytest in your project to perform all the unit testings, it provides a consistent pytest behavior.

If you have multiple tests running by pytest, if you use --cov-append to append this coverage data to coverage data from previous test runs.
