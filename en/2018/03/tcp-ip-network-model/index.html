The TCP/IP Five-Layer Network Model

Lucas Xu / 2018-03-09

Physical Layer

Physical layer represents the physical devices that interconnect computers, which includes the specifications for the networking cables and the connectors that join devices together along with specifications how signals are sent over these connections.

Data Link Layer (Network Interface, or Network Access Layer)

Data link layer is responsible for defining a common way of interpreting these signals so network devices can communicate.

The most common protocol is called Ethernet, even though wireless technologies are also very common in today’s world. The Ethernet standards also define a protocol responsible for getting data to nodes on the same network or link.

Network Layer (Internet Layer)

Network layer allows different networks to communicate with each other through devices known as router. A collection of networks connected together through routers, the most famous of these being the Internet.

The most common protocol used at this layer is known as Internet Protocol (IP). IP is the heart of the internet and most smaller networks around the world.

Transport Layer

Transport layer sorts out which client and server programs are supposed to get that data.

The most common protocol within this layer is called Transmission Control Protocol (TCP). Another transfer protocol used IP to get around is User Datagram Protocol (UDP). The difference between them is that TCP provides mechanisms to ensure that data is reliably delivered while UDP does not.

Application Layer

Lots of different protocols at this layer, and they are application-specific. The common ones are the protocols that allow you to browse the web or send/receive emails.

Summary

  • The physical layer is all about cabling, connectors and signals.
  • The data link layer is responsible for getting data across a single link.
  • The network layer is responsible for getting data delivered across a collection of networks.
  • The transport layer is responsible for ensuring that data gets to the right applications running on the nodes.
  • The application layer is the contents of itself.

Other Models

The seven-layer OSI Networking Model.

The biggest difference between the five-layer model and the OSI networking model is that the OSI model abstracts the application layer into three layers total.
