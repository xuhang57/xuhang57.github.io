什么是链表 (What is a Linked List?)

链表是有一连串结点（链表中的元素称为结点），每个结点由两部分组成，一个是储存数据元素的数据域，另一个是储存下一个结点地址的指针域。

A linked list is a linear data structure where each element is a separate object. It is a collection of nodes.

Each element (node) of a list is comprising of two items: the data and a reference to the next node (Singly linked list).

A doubly linked list is a list that has two two references, one to the next node and another previous node.

Another important type of a linked list is called a circular linked list, where last node of the list points back to the first node (or the head) of the list.

为什么使用链表 (Why use a Linked List?)

当时用数组存储数据时，我们必须事先定义数组长度。如果我们对数组大小不确定，必须把数组定义的足够大。但这样会造成内存浪费。另外，数组中的插入或者删除元素也会造成很大的时间开销。

但是链表失去了数组可以随机读取的优点，同时链表增加了结点的指针域，使得空间开销较大。

Pros:

One disadvantage of using arrays to store data is that arrays are static structures and therefore cannot be easily extended or reduced to fit the data set. Arrays are also too expensive to maintain new insertions and deletions.

Given the feature of the linked list, that the numbers of nodes in the list can grow and shrink on demand, any application which has to deal with an unknown number of objects will need to use a linked list.

  • A linked list saves memory.
  • Linked list nodes can live anywhere in the memory (an array requires a sequence of memory to be initiated).
  • Can be traversed in both forward and backward direction.

Cons:

One disadvantage of a linked list against an array is that it does not allow direct access to the individual elements. If you want to access a particular item then you have to start at at the head and follow the references until you get to that item.

Another disadvantage is that a linked list uses more memory compared with an array. We use extra 4 bytes (on a 32-bit CPU) to store a reference to the next node.

  • Linear look up time. (Many languages allow constant lookups in arrays).
  • Every node requires extra space for a previous pointer.
  • All operations require an extra pointer to be maintained.

链表特点 (Features)

头指针是指链表中的第一个结点的储存位置。为了简化链表的操作，有时会在链表的第一个结点之前设置一个新的结点，称为头结点。头结点的数据域可以不储存任何信息，也可以储存如线性表长度等附加信息，指针域储存指向第一个结点的指针。最后一个结点则指向Null

The last node has a reference to null. The entry point into a linked list is called a head of the list. It should be noted that head is not a separate node, but the reference to the first node. If the list is empty then the head is a null reference.

Linked List in Java (Singly)

private static class Node<T>
{
    private T data;
    private Node<T> next;

    public Node(T data, Node<T> next)
    {
         this.data = data;
         this.next = next;
    }
}

In Java, you can define an inner class. It is a member of its enclosing class and has access to other members (including private) of the outer class. And vise versa, the outer class can have a direct access to all members of the inner class.

An inner class can be declared private, public, protected, or package private. There are two kind of inner classes: static and non-static. A static inner class cannot refer directly to instance variables or methods methods defined in its outer class: it can use them only through an object reference.

A static java inner class cannot have instances. A non-static java inner class can have instances that belong to the outer class.

Linked List in Python (Singly)

class Node:
    def __init__(self, val):
        self.val = val
        self.next = None

    def getVal(self):
        return self.val

    def getNext(self):
        return self.next

    def setVal(self, new_val):
        self.val = new_val

    def setNext(self, new_next):
        self.next = new_next

>>> node1 = Node(12)
>>> node2 = Node(99)
>>> node3 = Node(37)

>>> node1.next = node2 # 12 -> 99
>>> node2.next = node3 # 99 -> 37

# the entire linkded list now looks like 12 -> 99 -> 37

Doubly Linked List in Python

class DoublyNode:
    def __init__(self, val):
        self.val = val
        self.next = None
        self.prev = None

时间/空间复杂度 (Time/Space Complexity)

  • Singly-Linked List

    • Time Complexity

      • Access: O(n)
      • Search: O(n)
      • Insertion: O(1) (Considering “Indexing” is a separate operation.)
      • Deletion: O(1)

    • Space Complexity: O(n)

  • Doubly-Linked List

    • Time Complexity
      • Access: O(n)
      • Search: O(n)
      • Insertion: O(1)
      • Deletion: O(1)
    • Space Complexity: O(n)

Some fun coding challenges

  • Delete a node from a linked list: singly and doubly.
  • Add two linked lists from left to right.
    • e.g. 1 -> 2 -> 3 + 4 -> 5 => 321 + 54 = 375
  • Add two linked lists from right to left.
